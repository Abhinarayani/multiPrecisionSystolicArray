🚀 QUICK START (3 STEPS)
═══════════════════════════════════════════════════════════════════════════════

Step 1: Create Quartus Project (2 minutes)
─────────────────────────────────────────────────────────────────────────────
  $ cd quartus
  $ quartus_sh -t setup_project.tcl

Step 2: Build Design (5-10 minutes)
─────────────────────────────────────────────────────────────────────────────
  $ quartus_map bitsys_de1
  $ quartus_fit bitsys_de1
  $ quartus_asm bitsys_de1

Step 3: Program & Test (5 minutes)
─────────────────────────────────────────────────────────────────────────────
  $ quartus_pgm -c "USB-Blaster [USB-0]" -m JTAG -o "P;bitsys_de1.sof"
  $ python host/bitsys_host.py COM3 1

📊 SPECIFICATIONS
═══════════════════════════════════════════════════════════════════════════════

COMMUNICATION
  ├─ Protocol:      115200 baud, 8N1 (8 data, no parity, 1 stop)
  ├─ Commands:      4 commands (0x01-0x04)
  ├─ Data Format:   Row-major 4×4 matrices (8-bit signed elements)
  ├─ Results:       Big-endian 32-bit signed integers
  └─ Bandwidth:     ~14 kB/s @ 115200 baud

PERFORMANCE
  ├─ Computation:   220 ns per 4×4 matrix (11 cycles @ 50 MHz)
  ├─ Throughput:    4.5M matrices/sec (compute limited)
  ├─ Serial Time:   ~8-9 ms per matrix (serial limited)
  ├─ Total Latency: 220 ns + serial overhead
  └─ Bottleneck:    Serial communication, not computation

HARDWARE
  ├─ FPGA:          Cyclone V (5CSEMA5F31C6)
  ├─ Area:          ~3500 LUTs (with clock gating)
  ├─ Power (idle):  ~200 mW (estimated)
  ├─ Power (active):~600 mW (estimated)
  ├─ Clock:         50 MHz (CLOCK_50)
  └─ Reset:         Active-low (KEY0)
